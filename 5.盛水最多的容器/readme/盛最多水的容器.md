

# 5. 盛最多水的容器 - 详细解题分析

## 一. 题目

给定一个长度为 n 的整数数组 `height`。有 n 条垂线，第 i 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

说明：你不能倾斜容器。

---

## 1. 示例

**示例 1：**
```
输入：[1,8,6,2,5,4,8,3,7]
输出：49
解释：图中垂直线代表输入数组 [1,8,6,2,5,4,8,3,7]。在此情况下，容器能够容纳水（表示为蓝色部分）的最大值为 49。
```

**示例 2：**
```
输入：height = [1,1]
输出：1
```

---

## 2. 提示

- `n == height.length`
- `2 <= n <= 10^5`
- `0 <= height[i] <= 10^4`

---

## 二. 解题思路

### 1. 问题分析

**目标**：找到两条垂线，使它们与 x 轴构成的容器能容纳最多的水

**关键约束**：
- 容器的面积由两条线之间的距离和较短那条线的高度决定
- 不能倾斜容器，意味着水面必须是水平的

**核心公式**：
```
面积 = 两条线之间的距离 × 两条线中较短的那条
area = (right - left) × min(height[left], height[right])
```

**形象理解**：
```
        │
        │                │
        │                │
        │   ═════════════│═══
        │   ║     水     ║
        │   ║            ║
────────┴───────────────────────────
        ↑                        ↑
      left                      right
        
面积 = (right - left) × min(height[left], height[right])
```

## 解法一：暴力枚举（Brute Force）

### 思路

枚举所有可能的线对，计算每个容器的面积，找出最大值。

```python
def maxArea(height: list[int]) -> int:
    """
    暴力枚举法
    
    时间复杂度：O(n²)
    空间复杂度：O(1)
    """
    n = len(height)
    max_water = 0
    
    # 枚举所有可能的线对
    for left in range(n):
        for right in range(left + 1, n):
            # 计算当前容器的水量
            water = (right - left) * min(height[left], height[right])
            max_water = max(max_water, water)
    
    return max_water
```

### 图解过程

```
数组：[1, 8, 6, 2, 5, 4, 8, 3, 7]

暴力枚举过程：

left=0, right=1: area = (1-0) × min(1,8) = 1
left=0, right=2: area = (2-0) × min(1,6) = 2
left=0, right=3: area = (3-0) × min(1,2) = 3
left=0, right=4: area = (4-0) × min(1,5) = 4
left=0, right=5: area = (5-0) × min(1,4) = 5
left=0, right=6: area = (6-0) × min(1,8) = 6
left=0, right=7: area = (7-0) × min(1,3) = 7
left=0, right=8: area = (8-0) × min(1,7) = 7
...
left=1, right=8: area = (8-1) × min(8,7) = 49  ← 最大值！

总共需要尝试 C(n,2) = n×(n-1)/2 次
```

### 优缺点

- 优点：简单直接，易于理解
- 缺点：**时间复杂度 O(n²)**，对于大数据集会超时

## 解法二：双指针法（推荐！）

### 核心思想

将左右指针分别放在数组的两端，每次移动**较短**的那条线，因为：
- 移动较长的线不会增加面积（受限于较短的线）
- 移动较短的线**可能**找到更高的线，从而增加面积

```python
def maxArea(height: list[int]) -> int:
    """
    双指针法（推荐）
    
    时间复杂度：O(n)
    空间复杂度：O(1)
    
    核心思想：
        1. 左右指针从两端开始
        2. 计算当前容器的面积
        3. 移动较短的那条线
        4. 重复直到左右指针相遇
    """
    left = 0                      # 左指针
    right = len(height) - 1       # 右指针
    max_water = 0                 # 最大水量
    
    while left < right:
        # 计算当前容器的水量
        water = (right - left) * min(height[left], height[right])
        max_water = max(max_water, water)
        
        # 移动较短的那条线（核心优化）
        if height[left] < height[right]:
            left += 1
        else:
            right -= 1
    
    return max_water
```

### 为什么移动较短的那条线？

```
┌─────────────────────────────────────────────────────────────┐
│                 为什么移动较短的那条线？                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  假设：height[left] < height[right]                        │
│                                                             │
│  当前面积 = (right - left) × height[left]                 │
│                                                             │
│  情况1：移动较长的线（right 向左移动）                      │
│  ───────────────────────────────────────────────────────  │
│  新距离 = right - left - 1                                 │
│  新高度 ≤ height[left]（因为新线不会比 left 的线高）       │
│  新面积 ≤ (right - left - 1) × height[left]               │
│           < (right - left) × height[left]                 │
│  结论：面积一定不会增加！                                   │
│                                                             │
│  情况2：移动较短的线（left 向右移动）                      │
│  ───────────────────────────────────────────────────────  │
│  新距离 = right - left - 1                                 │
│  新高度 可能 > height[left]（可能找到更高的线）           │
│  新面积 可能 > 当前面积                                     │
│  结论：有可能增加面积，值得尝试！                           │
│                                                             │
│  因此：总是移动较短的那条线                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 解法三：暴力优化（提前终止）

### 思路

暴力枚举，但在某些条件下提前终止搜索。

```python
def maxArea(height: list[int]) -> int:
    """
    暴力优化法
    
    时间复杂度：O(n²)（最坏情况）
    空间复杂度：O(1)
    
    优化：
        - 跳过不可能超过当前最大值的组合
        - 较短的线移动可以跳过更远的距离
    """
    n = len(height)
    max_water = 0
    
    for left in range(n):
        # 优化：如果最远的线形成的面积也不如当前最大值，直接结束
        max_possible = (n - 1 - left) * max(height[left:])
        if max_possible <= max_water:
            continue
        
        for right in range(left + 1, n):
            water = (right - left) * min(height[left], height[right])
            max_water = max(max_water, water)
    
    return max_water
```

### 优缺点

- 优点：在某些情况下比纯暴力法快
- 缺点：最坏情况时间复杂度仍是 O(n²)

## 三. 方法对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|-----------|-----------|------|------|
| **暴力枚举** | O(n²) | O(1) | 简单直观 | 对于大数据集超时 |
| **双指针法** | **O(n)** | O(1) | **最优解，推荐使用** | 需要理解为什么这样移动 |
| **暴力优化** | O(n²) | O(1) | 可提前终止 | 最坏情况仍是 O(n²) |

### 复杂度分析

```
暴力枚举：
  - 时间：O(n²) - 需要枚举所有 C(n,2) 种组合
  - 空间：O(1) - 只使用常数额外空间

双指针法：
  - 时间：O(n) - 每个元素最多被访问一次
  - 空间：O(1) - 只使用两个指针
```

### 为什么双指针法是 O(n)？

## 四. 边界情况处理

```python
def maxArea(height: list[int]) -> int:
    """盛最多水的容器 - 包含边界情况处理"""
    
    # 情况1：数组长度小于2（不应该发生，题目保证 n >= 2）
    if len(height) < 2:
        return 0
    
    # 情况2：所有高度为0
    # 例如：[0, 0, 0, 0]
    # 结果应该是0
    
    # 情况3：只有两条线
    # 例如：[1, 1]
    # 结果应该是 1 × min(1, 1) = 1
    
    # 情况4：包含重复的高度
    # 例如：[1, 1, 1, 1]
    # 结果应该是 3 × 1 = 3
    
    # 双指针算法本身已经处理了所有边界情况
    left = 0
    right = len(height) - 1
    max_water = 0
    
    while left < right:
        water = (right - left) * min(height[left], height[right])
        max_water = max(max_water, water)
        
        if height[left] < height[right]:
            left += 1
        else:
            right -= 1
    
    return max_water
```

## 五. 小贴士

### 1. 为什么双指针从两端开始？

```
┌─────────────────────────────────────────────────────────────┐
│                 为什么从两端开始？                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  原因1：初始时距离最大                                       │
│  ───────────────────────────────────────────────────────  │
│  - 从两端开始，初始距离是 n-1（最大可能距离）               │
│  - 即使高度很低，也可能因为距离大而成为最优解               │
│                                                             │
│  原因2：全面搜索                                            │
│  ───────────────────────────────────────────────────────  │
│  - 从两端开始，保证能遍历所有可能的线对                     │
│  - 如果从相邻开始，可能会漏掉最优解                        │
│                                                             │
│  原因3：配合移动策略                                        │
│  ───────────────────────────────────────────────────────  │
│  - 从两端开始，使得"移动较短的那条线"策略有意义            │
│  - 因为这样可以系统地探索所有可能的组合                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 六. 完整代码实现

### Python（双指针法 - 最优解）

```python
from typing import List

class Solution:
    def maxArea(self, height: List[int]) -> int:
        """
        盛最多水的容器 - 双指针法
        
        时间复杂度：O(n)
        空间复杂度：O(1)
        
        算法步骤：
            1. 初始化左右指针在数组两端
            2. 计算当前指针形成的容器面积
            3. 更新最大面积
            4. 移动较短的那条线
            5. 重复直到左右指针相遇
            
        Args:
            height: 整数数组，表示每条线的高度
            
        Returns:
            容器能储存的最大水量
        """
        left = 0                      # 左指针指向数组开头
        right = len(height) - 1       # 右指针指向数组结尾
        max_water = 0                 # 最大水量
        
        # 当左指针小于右指针时循环
        while left < right:
            # 计算当前容器的水量
            # 宽度 = 右指针 - 左指针
            # 高度 = 两指针中较短的那条
            width = right - left
            current_height = min(height[left], height[right])
            water = width * current_height
            
            # 更新最大水量
            max_water = max(max_water, water)
            
            # 移动较短的那条线（核心优化）
            # 这样可以有机会找到更高的线
            if height[left] < height[right]:
                left += 1      # 左指针右移
            else:
                right -= 1     # 右指针左移
        
        return max_water
```

### C++（双指针法）

```cpp
#include <vector>
#include <algorithm>

class Solution {
public:
    /**
     * 盛最多水的容器 - 双指针法
     * 
     * 时间复杂度：O(n)
     * 空间复杂度：O(1)
     */
    int maxArea(std::vector<int>& height) {
        int left = 0;                          // 左指针
        int right = height.size() - 1;         // 右指针
        int max_water = 0;                     // 最大水量
        
        while (left < right) {
            // 计算当前容器的水量
            int width = right - left;
            int current_height = std::min(height[left], height[right]);
            int water = width * current_height;
            
            // 更新最大水量
            max_water = std::max(max_water, water);
            
            // 移动较短的那条线
            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }
        
        return max_water;
    }
};
```

| 要点 | 说明 |
|------|------|
| **问题本质** | 在数组中找到两点，使 `(j-i) × min(height[i], height[j])` 最大 |
| **核心策略** | 双指针从两端开始，每次移动较短的那条线 |
| **为什么这样移动** | 移动较长的线，新面积一定不会增加；移动较短的线，可能增加 |
| **时间复杂度** | O(n)（因为每个元素最多被访问一次） |
| **空间复杂度** | O(1)（只使用两个指针） |

### 一句话总结

盛最多水的容器是一道经典的**双指针**问题，核心思想是从数组两端开始，**每次移动较短的那条线**，这样可以在 O(n) 时间复杂度内找到最大面积。
