# 16. 除自身以外数组的乘积 - 详细解题分析

## 一. 题目

给你一个整数数组 `nums`，返回数组 `answer`，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的乘积。

题目数据保证数组 `nums` 之中任意元素的全部前缀元素和后缀的乘积都在 32 位整数范围内。

请不要使用除法，且在 O(n) 时间复杂度内完成此题。

### 示例

**示例 1：**
```
输入: nums = [1,2,3,4]
输出: [24,12,8,6]
解释:
answer[0] = 2*3*4 = 24
answer[1] = 1*3*4 = 12
answer[2] = 1*2*4 = 8
answer[3] = 1*2*3 = 6
```

**示例 2：**
```
输入: nums = [-1,1,0,-3,3]
输出: [0,0,9,0,0]
解释:
answer[0] = 1*0*-3*3 = 0
answer[1] = -1*0*-3*3 = 0
answer[2] = -1*1*-3*3 = 9
answer[3] = -1*1*0*3 = 0
answer[4] = -1*1*0*-3 = 0
```

### 提示

- `2 <= nums.length <= 10^5`
- `-30 <= nums[i] <= 30`
- 输入保证数组 `answer[i]` 在 32 位整数范围内

**进阶**：
- 你可以在 O(1) 的额外空间复杂度内完成这个题目吗？（出于对空间复杂度分析的目的，输出数组不被视为额外空间。）

---

## 二. 解题思路

### 1. 问题分析

**目标**：计算每个位置除自身外所有元素的乘积

**关键约束**：
- 不能使用除法（避免除零问题）
- 需要 O(n) 时间复杂度
- 进阶：O(1) 额外空间（输出数组不计入）

**核心思想**：
- **前缀后缀法**：利用前缀积和后缀积的思想
- **双遍历法**：先计算左侧乘积，再计算右侧乘积
- 不能简单使用总乘积除以当前元素（禁止除法，且可能有零）

### 2. 关键洞察

```
对于数组 nums = [1, 2, 3, 4]：

answer[0] = nums[1] * nums[2] * nums[3] = 2 * 3 * 4 = 24
answer[1] = nums[0] * nums[2] * nums[3] = 1 * 3 * 4 = 12
answer[2] = nums[0] * nums[1] * nums[3] = 1 * 2 * 4 = 8
answer[3] = nums[0] * nums[1] * nums[2] = 1 * 2 * 3 = 6

观察规律：
  answer[i] = (nums[0] * nums[1] * ... * nums[i-1]) * (nums[i+1] * ... * nums[n-1])
            =   左侧所有元素的乘积       *   右侧所有元素的乘积
```

## 三. 解法一：前缀后缀法（两次遍历）【推荐】

### 1. 算法思想

分两步计算：
- **第一步（从左到右）**：计算每个位置的左侧累积乘积
- **第二步（从右到左）**：将右侧累积乘积乘入结果

### 2. 算法步骤

```
1. 初始化结果数组 answer，长度为 n
2. 第一次遍历（从左到右）：
   a. left = 1
   b. 对于 i = 0 到 n-1：
      - answer[i] = left（表示 nums[0..i-1] 的乘积）
      - left = left * nums[i]（更新左侧乘积）
3. 第二次遍历（从右到左）：
   a. right = 1
   b. 对于 i = n-1 到 0：
      - answer[i] = answer[i] * right（乘以右侧的累积乘积）
      - right = right * nums[i]（更新右侧乘积）
4. 返回 answer
```

### 3. 代码实现

#### Python实现

```python
from typing import List

class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        """
        除自身以外数组的乘积 - 前缀后缀法

        时间复杂度：O(n)
        空间复杂度：O(1) - 不计入输出数组

        算法思想：
          1. 第一步：从左到右，计算每个位置的左侧累积乘积
          2. 第二步：从右到左，乘以右侧累积乘积
        """
        n = len(nums)
        if n == 0:
            return []

        answer = [0] * n

        left = 1
        for i in range(n):
            answer[i] = left
            left *= nums[i]

        right = 1
        for i in range(n - 1, -1, -1):
            answer[i] *= right
            right *= nums[i]

        return answer
```

#### C++实现

```cpp
#include <vector>
using namespace std;

class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        int n = nums.size();
        if (n == 0) {
            return {};
        }

        vector<int> answer(n, 1);

        int left = 1;
        for (int i = 0; i < n; i++) {
            answer[i] = left;
            left *= nums[i];
        }

        int right = 1;
        for (int i = n - 1; i >= 0; i--) {
            answer[i] *= right;
            right *= nums[i];
        }

        return answer;
    }
};
```

```

第二步：从右到左，乘以右侧累积乘积
```

### 5. 复杂度分析

```
时间复杂度：O(n)
  - 第一次遍历：n 次操作
  - 第二次遍历：n 次操作
  - 总共 2n 次操作

空间复杂度：O(1)
  - 只使用了常数个额外变量（left, right, answer）
  - 输出数组 answer 不计入额外空间（题目说明）
  - 满足进阶要求
```

## 四. 解法二：两次遍历（先计算所有乘积）

### 1. 算法思想

先计算整个数组的乘积，然后对每个位置，除以该位置的元素。

**问题**：题目要求不能使用除法！

### 2. 算法步骤（错误的示范）

```
1. 计算所有元素的乘积：product = 1
2. 对于 i = 0 到 n-1：
   - answer[i] = product / nums[i]  # 错误：不能使用除法
3. 返回 answer
```

### 3. 为什么不能使用除法？

```
情况1：包含零
  nums = [1, 2, 0, 4]
  如果使用除法：
    product = 1 * 2 * 0 * 4 = 0
    answer[0] = 0 / 1 = 0  ✓
    answer[1] = 0 / 2 = 0  ✓
    answer[2] = 0 / 0     ✗ 除零错误！
    answer[3] = 0 / 4 = 0  ✓

情况2：零的个数大于1
  nums = [0, 1, 2]
  answer[0] = 1 * 2 / 0   ✗ 除零错误！

所以必须使用前缀后缀法，避免除法。
```

## 五. 解法三：包含零的特殊处理

### 1. 算法思想

如果数组中只有一个零，结果中只有该位置是非零的（等于所有非零元素的乘积）。
如果数组中有多个零，所有位置的结果都是零。

### 2. 代码实现

```python
class Solution:
    def productExceptSelf_withZero(self, nums: List[int]) -> List[int]:
        """
        包含零的特殊处理（不推荐，仅作理解）

        时间复杂度：O(n)
        空间复杂度：O(1)
        """
        n = len(nums)
        zero_count = nums.count(0)

        if zero_count == 0:
            # 没有零，可以使用除法（但题目要求不能使用）
            total_product = 1
            for num in nums:
                total_product *= num
            return [total_product // num for num in nums]

        elif zero_count == 1:
            # 只有一个零，只有零的位置是非零的
            result = [0] * n
            product_without_zero = 1
            for num in nums:
                if num != 0:
                    product_without_zero *= num

            for i, num in enumerate(nums):
                if num == 0:
                    result[i] = product_without_zero
                    break
            return result

        else:
            # 有多个零，所有结果都是零
            return [0] * n
```

**说明**：虽然这个方法可以工作，但题目明确要求不能使用除法，所以前缀后缀法是唯一正确的解法。

## 七. 深入理解前缀后缀法

### 1. 前缀和与前缀积

```
前缀积 prefix[i] = nums[0] * nums[1] * ... * nums[i-1]

对于 nums = [1, 2, 3, 4]：
  prefix[0] = 1（单位元，空前缀）
  prefix[1] = 1
  prefix[2] = 1 * 2 = 2
  prefix[3] = 1 * 2 * 3 = 6
  prefix[4] = 1 * 2 * 3 * 4 = 24
```

### 2. 后缀和与后缀积

```
后缀积 suffix[i] = nums[i+1] * nums[i+2] * ... * nums[n-1]

对于 nums = [1, 2, 3, 4]：
  suffix[3] = 1（单位元，空后缀）
  suffix[2] = 4
  suffix[1] = 3 * 4 = 12
  suffix[0] = 2 * 3 * 4 = 24
```

### 3. 关系

```
answer[i] = prefix[i] * suffix[i]

对于 nums = [1, 2, 3, 4]：
  answer[0] = prefix[0] * suffix[0] = 1 * 24 = 24
  answer[1] = prefix[1] * suffix[1] = 1 * 12 = 12
  answer[2] = prefix[2] * suffix[2] = 2 * 4 = 8
  answer[3] = prefix[3] * suffix[3] = 6 * 1 = 6
```

### 4. 空间优化

```
朴素的两次遍历：
  - 需要额外的 prefix 和 suffix 数组
  - 空间复杂度：O(n)

优化的两次遍历：
  - 只使用一个结果数组
  - 第一次遍历填入左侧乘积
  - 第二次遍历乘以右侧乘积
  - 空间复杂度：O(1)（不计输出数组）
```

### 5. 代码优化细节

```python
# 第一步：从左到右
left = 1
for i in range(n):
    answer[i] = left  # 填入左侧乘积
    left *= nums[i]  # 更新左侧累积

# 第二步：从右到左
right = 1
for i in range(n - 1, -1, -1):
    answer[i] *= right  # 乘以右侧乘积
    right *= nums[i]  # 更新右侧累积
```

## 九. 完整代码汇总

### Python实现（最终版本）

```python
from typing import List


class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        """
        除自身以外数组的乘积 - 前缀后缀法

        时间复杂度：O(n)
        空间复杂度：O(1) - 不计入输出数组

        算法思想：
          1. 第一步：从左到右，计算每个位置的左侧累积乘积
          2. 第二步：从右到左，乘以右侧累积乘积

        为什么不能使用除法？
          - 数组中可能有零，除以零会出错
          - 如果有多个零，除以零完全没有意义
          - 前缀后缀法天然避免除法，适用范围更广

        进阶要求：
          - 输出数组不计入额外空间
          - 只使用常数个额外变量
          - 满足 O(1) 空间复杂度
        """
        n = len(nums)
        if n == 0:
            return []

        answer = [0] * n

        left = 1
        for i in range(n):
            answer[i] = left
            left *= nums[i]

        right = 1
        for i in range(n - 1, -1, -1):
            answer[i] *= right
            right *= nums[i]

        return answer
```

```cpp
        }

    }
};
```

### 为什么前缀后缀法最优？

```
1. 时间复杂度 O(n)：
   - 只需要两次遍历
   - 每个元素最多访问两次

2. 空间复杂度 O(1)：
   - 只使用常数个额外变量
   - 输出数组不计入空间
   - 满足进阶要求

3. 不使用除法：
   - 避免除零错误
   - 适用范围更广
   - 代码更健壮

4. 代码简洁：
   - 核心逻辑只有两行代码
   - 易于理解和实现
```

## 十二. 面试要点

```
面试官可能会问的问题：

1. 为什么不能使用除法？
   - 如果数组中有零，除以零会导致错误
   - 如果有多个零，结果全是零，无法用除法表示
   - 前缀后缀法天然避免除法，更通用

2. 如何在 O(1) 空间内完成？
   - 只使用一个结果数组存储左侧乘积
   - 第二次遍历时，从右到左累积右侧乘积
   - 将右侧乘积直接乘入结果数组

3. 前缀后缀法的核心思想是什么？
   - 将问题分解为"左侧"和"右侧"两部分
   - 分别计算累积乘积
   - 合并结果

4. 时间复杂度为什么是 O(n)？
   - 第一次遍历：n 次操作
   - 第二次遍历：n 次操作
   - 总共 2n 次操作，即 O(n)

5. 如何处理边界情况？
   - 空数组：返回空数组
   - 单个元素：返回 [1]
   - 包含零：自动处理，零的位置的答案是所有非零元素的乘积
```

## 十三. 总结

### 核心算法

```
前缀后缀法（优化版）：
  1. 第一步：从左到右，计算左侧累积乘积，存入结果数组
  2. 第二步：从右到左，将右侧累积乘积乘入结果数组
  3. 返回结果数组

关键点：
  - 不能使用除法
  - O(n) 时间复杂度
  - O(1) 空间复杂度（不计输出数组）
  - 两次遍历技巧
```

### 关键点总结

1. **前缀后缀法**：利用累积乘积的思想
2. **两次遍历**：先左到右，再右到左
3. **O(1) 空间**：只使用常数个变量，输出数组不计入
4. **避免除法**：自然处理零的情况

### 最终建议

```

  ✅ 代码简洁高效
```
