
# 3. 最长连续序列 

## 一. 题目

给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 **O(n)** 的算法解决此问题。

### 示例

**示例 1：**
```
输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]，它的长度为 4。
```

**示例 2：**
```
输入：nums = [0,3,7,2,5,8,4,6,0,1]
输出：9
```

**示例 3：**
```
输入：nums = [1,0,1,2]
输出：3
```

### 提示

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`

---

## 二. 解题思路

### 1. 问题分析

**目标**：找出数组中最长的连续数字序列的长度

**关键点**：
- 不要求元素在原数组中位置连续，只要求数值连续
- 时间复杂度必须为 O(n)
- 需要正确处理重复元素和边界情况

**核心思想**：对于每个数字 x，检查 x+1、x+2、x+3... 是否存在，找到最长的连续序列

---

### 解法一：排序法

#### 思路

将数组排序后，遍历数组找出最长的连续序列。

```python
def longestConsecutive(self, nums: List[int]) -> int:
    """
    排序法解决最长连续序列
    
    时间复杂度：O(n log n)
    空间复杂度：O(1) 或 O(n)（取决于排序算法）
    """
    # 处理空数组情况
    if not nums:
        return 0
    
    # 对数组排序
    nums_sort = sorted(nums)
    
    # 初始化计数器和最大长度
    count = 1      # 当前连续序列的长度
    max_count = 1  # 最长连续序列的长度
    
    # 遍历排序后的数组
    for i, num in enumerate(nums_sort):
        # 如果不是最后一个元素
        if i != len(nums_sort) - 1:
            # 如果下一个元素是连续的数字
            if nums_sort[i + 1] == num + 1:
                count += 1
            # 如果下一个元素与当前元素相等（重复）
            elif nums_sort[i + 1] == num:
                continue  # 跳过，继续检查
            # 如果下一个元素与当前元素不连续
            else:
                count = 1  # 重置计数器
        
        # 更新最大长度
        if max_count < count:
            max_count = count
    
    return max_count
```

#### 图解过程

```
数组：[100, 4, 200, 1, 3, 2]
排序后：[1, 2, 3, 4, 100, 200]

步骤1：i=0, num=1
  nums[i+1]=2, 1+1=2 ✓ 连续
  count = 2
  max_count = 2

步骤2：i=1, num=2
  nums[i+1]=3, 2+1=3 ✓ 连续
  count = 3
  max_count = 3

步骤3：i=2, num=3
  nums[i+1]=4, 3+1=4 ✓ 连续
  count = 4
  max_count = 4

步骤4：i=3, num=4
  nums[i+1]=100, 4+1=5 ✗ 不连续
  count = 1

步骤5：i=4, num=100
  nums[i+1]=200, 100+1=101 ✗ 不连续
  count = 1

步骤6：i=5, num=200
  最后一个元素，不处理

最终结果：max_count = 4
```

#### 优缺点

- ✅ 思路清晰，易于理解
- ✅ 代码相对简单
- ❌ **时间复杂度 O(n log n)，不满足题目要求**
- ❌ 排序后改变了元素顺序（虽然题目不需要保留，但不如哈希表法优雅）

### 解法二：哈希表法（推荐！）

#### 核心思想

利用哈希表（集合）的 O(1) 查找特性，**只从序列的起点开始查找**，避免重复查找。

**关键洞察**：只从连续序列的**最小值**开始查找，可以避免大量重复查找。

```python
def longestConsecutive(self, nums: List[int]) -> int:
    """
    哈希表法解决最长连续序列（最优解）
    
    时间复杂度：O(n)
    空间复杂度：O(n)
    """
    # 处理空数组情况
    if not nums:
        return 0
    
    # 将数组转换为集合（哈希表）
    num_set = set(nums)
    
    # 最长连续序列的长度
    max_count = 0
    
    # 遍历数组中的每个数字
    for num in num_set:
        # 只从连续序列的起点开始查找
        # 判断是否是起点：num-1 不在集合中
        if num - 1 not in num_set:
            # 从起点开始，统计连续序列的长度
            current_num = num
            current_count = 1
            
            # 查找连续的后续数字
            while current_num + 1 in num_set:
                current_num += 1
                current_count += 1
            
            # 更新最大长度
            max_count = max(max_count, current_count)
    
    return max_count
```

#### 图解过程

```
数组：[100, 4, 200, 1, 3, 2]
集合：{100, 4, 200, 1, 3, 2}

遍历过程：

1. num=100
   - 100-1=99 不在集合中？是起点！
   - 查找 101, 102, ... （都不在）
   - current_count = 1

2. num=4
   - 4-1=3 在集合中！不是起点，跳过

3. num=200
   - 200-1=199 不在集合中？是起点！
   - 查找 201, 202, ... （都不在）
   - current_count = 1

4. num=1 ✓ 起点
   - 1-1=0 不在集合中
   - 查找 2（在集合中）→ count=2
   - 查找 3（在集合中）→ count=3
   - 查找 4（在集合中）→ count=4
   - 查找 5（不在集合中）
   - current_count = 4
   - max_count = 4

5. num=3
   - 3-1=2 在集合中！不是起点，跳过

6. num=2
   - 2-1=1 在集合中！不是起点，跳过

最终结果：max_count = 4
```

#### 为什么哈希表法是 O(n)？

```
普通方法的问题：
- 对每个数字都查找整个连续序列
- 例如：1, 2, 3, 4, 5
  - 从 1 开始：查找 1,2,3,4,5（5次）
  - 从 2 开始：查找 2,3,4,5（4次）← 重复！
  - 从 3 开始：查找 3,4,5（3次）← 重复！
  - 从 4 开始：查找 4,5（2次）← 重复！
  - 从 5 开始：查找 5（1次）← 重复！
  - 总共：5+4+3+2+1 = 15 次查找

哈希表法的优化：
- 只从起点开始查找
- 例如：1, 2, 3, 4, 5
  - 从 1 开始：查找 1,2,3,4,5（5次）
  - 2,3,4,5 都跳过（因为不是起点）
  - 总共：5 次查找

结论：每个元素最多被访问 2 次（一次判断是否为起点，一次在 while 循环中）
      时间复杂度：O(n)
```

### 解法三：并查集

#### 思路

使用并查集（Union-Find）将连续的元素合并到同一个集合中，然后统计每个集合的大小。

```python
class UnionFind:
    """并查集"""
    
    def __init__(self, nums: List[int]):
        """初始化并查集"""
        # 每个元素的父节点指向自己
        self.parent = {num: num for num in nums}
        # 每个集合的大小
        self.size = {num: 1 for num in nums}
    
    def find(self, x: int) -> int:
        """查找根节点（路径压缩）"""
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    
    def union(self, x: int, y: int):
        """合并两个集合"""
        root_x = self.find(x)
        root_y = self.find(y)
        
        if root_x != root_y:
            # 按秩合并：小集合合并到大集合
            if self.size[root_x] < self.size[root_y]:
                root_x, root_y = root_y, root_x
            
            self.parent[root_y] = root_x
            self.size[root_x] += self.size[root_y]

def longestConsecutive(self, nums: List[int]) -> int:
    """
    并查集法解决最长连续序列
    
    时间复杂度：O(n α(n)) ≈ O(n)
    空间复杂度：O(n)
    """
    if not nums:
        return 0
    
    uf = UnionFind(nums)
    
    for num in nums:
        # 只合并相邻的元素
        if num + 1 in uf.parent:
            uf.union(num, num + 1)
    
    return max(uf.size.values())
```

#### 图解过程

```
数组：[100, 4, 200, 1, 3, 2]

步骤1：初始化
  每个元素都是独立的集合
  parent: {100:100, 4:4, 200:200, 1:1, 3:3, 2:2}
  size: {100:1, 4:1, 200:1, 1:1, 3:1, 2:1}

步骤2：合并相邻元素
  遍历 num=100: 101 不在，跳过
  遍历 num=4: 5 不在，跳过
  遍历 num=200: 201 不在，跳过
  遍历 num=1: 2 在 → union(1, 2)
    parent: {1:1, 2:1, 3:3, 4:4, 100:100, 200:200}
    size: {1:2, 3:3, 4:4, 100:100, 200:200}
  遍历 num=3: 4 在 → union(3, 4)
    parent: {1:1, 2:1, 3:1, 4:1, 100:100, 200:200}
    size: {1:4, 4:4, 100:100, 200:200}
  遍历 num=2: 3 在 → union(2, 3)
    但 2 和 3 已经在同一个集合中，无需合并

步骤3：统计最大集合大小
  max(size) = 4

最终结果：4
```

## 三. 方法对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|-----------|-----------|------|------|
| **排序法** | O(n log n) | O(1) | 思路清晰 | 不满足 O(n) 要求 |
| **哈希表法** | **O(n)** | O(n) | 最优解，代码简洁 | 需要额外空间 |
| **并查集** | O(n α(n)) ≈ O(n) | O(n) | 通用性强 | 代码复杂 |

### 推荐

**哈希表法**是这道题的最优解，原因：
1. 时间复杂度 O(n)，满足题目要求
2. 代码简洁，易于理解和实现
3. 空间换时间，牺牲 O(n) 空间换取 O(n) 时间

## 四. 小贴士

### 1. 为什么只从起点开始查找？

```
数组：[1, 2, 3, 4, 5]

错误方法（每个数都查找）：
- 从 1 开始：查找 1,2,3,4,5
- 从 2 开始：查找 2,3,4,5（重复）
- 从 3 开始：查找 3,4,5（重复）
- ...
总计：15 次查找

正确方法（只从起点开始）：
- 从 1 开始：查找 1,2,3,4,5
- 2,3,4,5 都跳过（因为前一个数存在）
总计：5 次查找
```

### 2. 如何判断一个数是否是起点？

```python
# 一个数是连续序列的起点，当且仅当它的前一个数不在集合中
if num - 1 not in num_set:
    # num 是起点
```

### 3. 边界情况处理

```python
# 空数组
if not nums:
    return 0

# 单个元素
if len(nums) == 1:
    return 1

# 全是重复元素
nums = [1, 1, 1, 1]  # 结果应该是 1

# 负数
nums = [-3, -2, -1, 0, 1]  # 结果应该是 5
```

### 4. 复杂度分析详解

**哈希表法为什么是 O(n)？**

```
每个元素最多被访问常数次：
1. 第一次：在 for 循环中判断是否为起点
2. 第二次：在 while 循环中被访问（如果它是起点）

总时间复杂度：O(n)
总空间复杂度：O(n)（用于存储集合）

对比排序法：
- 排序时间：O(n log n)
- 遍历时间：O(n)
- 总时间：O(n log n)
```

### 5. 哈希表 vs 排序法的选择

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| **面试时** | 哈希表法 | 展示对时间复杂度的追求 |
| **需要原数组顺序** | 哈希表法 | 不改变数组 |
| **内存受限** | 排序法 | O(1) 空间 |
| **大数据集** | 哈希表法 | O(n) 时间更优 |

## 五. 完整代码实现

### Python（哈希表法 - 最优解）

```python
from typing import List

class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        """
        哈希表法：时间复杂度 O(n)，空间复杂度 O(n)
        
        核心思想：只从连续序列的起点开始查找，避免重复查找
        
        Args:
            nums: 未排序的整数数组
            
        Returns:
            最长连续序列的长度
        """
        # 处理空数组
        if not nums:
            return 0
        
        # 将数组转换为集合
        num_set = set(nums)
        
        # 最长连续序列的长度
        max_count = 0
        
        # 遍历数组中的每个数字
        for num in num_set:
            # 只从连续序列的起点开始查找
            # 判断是否是起点：num-1 不在集合中
            if num - 1 not in num_set:
                # 从起点开始，统计连续序列的长度
                current_num = num
                current_count = 1

        return max_count
```

### C++（哈希表法）

```cpp
#include <vector>
#include <unordered_set>
#include <algorithm>

class Solution {
public:
    /**
     * 哈希表法解决最长连续序列
     * 
     * 时间复杂度：O(n)
     * 空间复杂度：O(n)
     */
    int longestConsecutive(std::vector<int>& nums) {
        // 处理空数组
        if (nums.empty()) {
            return 0;
        }
        
        // 将数组转换为 unordered_set（哈希表）
        std::unordered_set<int> num_set(nums.begin(), nums.end());
        
        // 最长连续序列的长度
        int max_count = 0;
        
        // 遍历数组中的每个数字
        for (int num : num_set) {
            // 只从连续序列的起点开始查找
            // 判断是否是起点：num-1 不在集合中
            if (num_set.find(num - 1) == num_set.end()) {
                // 从起点开始，统计连续序列的长度
                int current_num = num;
                int current_count = 1;
                
                // 查找连续的后续数字
                while (num_set.find(current_num + 1) != num_set.end()) {
                    current_num++;
                    current_count++;
                }
                
                // 更新最大长度
                max_count = std::max(max_count, current_count);
            }
        }
        
        return max_count;
    }
};
```
