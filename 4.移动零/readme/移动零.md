
# 4. 移动零 - 详细解题分析

## 一. 题目

给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。

请注意，**必须在不复制数组的情况下原地对数组进行操作**。

### 示例

**示例 1：**
```
输入: nums = [0,1,0,3,12]
输出: [1,3,12,0,0]
```

**示例 2：**
```
输入: nums = [0]
输出: [0]
```

### 提示

- `1 <= nums.length <= 10^4`
- `-2^31 <= nums[i] <= 2^31 - 1`

### 进阶要求

你能**尽量减少完成的操作次数**吗？

---

## 二. 解题思路

### 1. 问题分析

**目标**：将数组中的所有 0 移动到末尾，同时保持非零元素的相对顺序

**关键约束**：
- **原地操作**：不能使用额外的数组
- **保持顺序**：非零元素的相对位置不能改变
- **减少操作**：进阶要求，尽量减少交换或移动次数

**核心思想**：使用**双指针**，将非零元素"挤压"到前面

---

### 解法一：冒泡排序法（简单但低效）

#### 思路

使用冒泡排序的思想，将所有 0 不断向后"冒泡"。

```python
def moveZeroes(nums: list[int]) -> None:
    """
    移动零 - 冒泡排序法
    
    时间复杂度：O(n²)
    空间复杂度：O(1)
    
    说明：
        - 通过不断交换将0向后移动
        - 代码简单但效率低
    """
    n = len(nums)
    
    # 外层循环：需要移动多少个0
    for i in range(n):
        # 内层循环：将最大的0冒泡到最后
        for j in range(0, n - i - 1):
            # 如果当前元素是0，下一个元素不是0，则交换
            if nums[j] == 0 and nums[j + 1] != 0:
                nums[j], nums[j + 1] = nums[j + 1], nums[j]
```

#### 图解过程

```
数组：[0, 1, 0, 3, 12]

第1轮冒泡：
  [0, 1, 0, 3, 12] → 位置0是0，位置1不是0 → 交换
  [1, 0, 0, 3, 12] → 位置1是0，位置2是0 → 不交换
  [1, 0, 0, 3, 12] → 位置2是0，位置3不是0 → 交换
  [1, 0, 3, 0, 12] → 位置3是0，位置4不是0 → 交换
  [1, 0, 3, 12, 0]

第2轮冒泡：
  [1, 0, 3, 12, 0] → 位置0不是0 → 不交换
  [1, 0, 3, 12, 0] → 位置1是0，位置2不是0 → 交换
  [1, 3, 0, 12, 0] → 位置2是0，位置3不是0 → 交换
  [1, 3, 12, 0, 0]

第3轮冒泡：
  [1, 3, 12, 0, 0] → 所有0已在最后，无需交换

最终结果：[1, 3, 12, 0, 0]
```

#### 优缺点

- 优点：代码简单，容易理解
- 缺点：**时间复杂度 O(n²)**，操作次数过多

### 解法二：双指针法（推荐！）

#### 思路

使用两个指针：
- **慢指针 (slow)**：指向下一个非零元素应该存放的位置
- **快指针 (fast)**：用于遍历数组

**核心思想**：将非零元素"挤压"到前面，然后剩余位置填 0。

```python
def moveZeroes(nums: list[int]) -> None:
    """
    移动零 - 双指针法（推荐）
    
    时间复杂度：O(n)
    空间复杂度：O(1)
    
    说明：
        - 使用两个指针，一个用于遍历，一个用于记录位置
        - 只需遍历一次数组
        - 非零元素被"挤压"到前面
    """
    n = len(nums)
    slow = 0  # 慢指针：指向下一个非零元素应该存放的位置
    
    # 快指针遍历数组
    for fast in range(n):
        # 如果当前元素非零
        if nums[fast] != 0:
            # 将非零元素移动到慢指针位置
            nums[slow] = nums[fast]
            
            # 如果慢指针和快指针不在同一位置
            # （即这个位置原本是0），则需要处理
            if slow != fast:
                # 将原位置设为0
                nums[fast] = 0
            
            # 慢指针前移，指向下一个空位
            slow += 1
```

#### 图解过程

```
数组：[0, 1, 0, 3, 12]

初始状态：
  slow = 0
  fast = 0
  [0, 1, 0, 3, 12]
   ↑
  slow, fast

步骤1：fast=0, nums[0]=0（非零？）
  否，跳过，fast++

步骤2：fast=1, nums[1]=1（非零？）
  是！
  nums[slow]=nums[1] → nums[0]=1
  slow=0, fast=1，位置不同，nums[1]=0
  slow++ → slow=1
  [1, 0, 0, 3, 12]
        ↑
      slow

步骤3：fast=2, nums[2]=0（非零？）
  否，跳过，fast++

步骤4：fast=3, nums[3]=3（非零？）
  是！
  nums[slow]=nums[3] → nums[1]=3
  slow=1, fast=3，位置不同，nums[3]=0
  slow++ → slow=2
  [1, 3, 0, 0, 12]
           ↑
         slow

步骤5：fast=4, nums[4]=12（非零？）
  是！
  nums[slow]=nums[4] → nums[2]=12
  slow=2, fast=4，位置不同，nums[4]=0
  slow++ → slow=3
  [1, 3, 12, 0, 0]
              ↑
            slow

遍历结束，slow=3
slow 之后的位置（索引3,4）已经是0
无需额外操作

最终结果：[1, 3, 12, 0, 0]
```

#### 为什么双指针法高效？

```
操作次数分析：

冒泡排序法：
  - 外层循环：n 次
  - 内层循环：平均 n/2 次
  - 总操作次数：O(n²)
  - 示例：100个元素，大约需要 100×50 = 5000 次操作

双指针法：
  - 只遍历一次数组：n 次
  - 每个元素最多移动一次
  - 总操作次数：O(n)
  - 示例：100个元素，只需要 100 次操作

性能提升：50倍！
```

### 解法三：交换法（进一步优化）

#### 思路

与双指针法类似，但只在需要时才交换位置。

```python
def moveZeroes(nums: list[int]) -> None:
    """
    移动零 - 交换法
    
    时间复杂度：O(n)
    空间复杂度：O(1)
    
    说明：
        - 使用双指针交换非零元素到前面
        - 比标准双指针法更简洁
    """
    n = len(nums)
    left = 0  # 左指针：指向下一个非零元素应该存放的位置
    
    for right in range(n):
        # 如果当前元素非零
        if nums[right] != 0:
            # 交换左右指针位置的元素
            nums[left], nums[right] = nums[right], nums[left]
            left += 1
```

#### 图解过程

```
数组：[0, 1, 0, 3, 12]

初始状态：
  left = 0
  right = 0
  [0, 1, 0, 3, 12]
   ↑     ↑
  left right

步骤1：right=0, nums[0]=0
  非零？否，right++

步骤2：right=1, nums[1]=1
  非零？是！
  交换 nums[left] 和 nums[right]
  nums[0] 和 nums[1] 交换
  [1, 0, 0, 3, 12]
       ↑  ↑
     left right
  left++ → left=1

步骤3：right=2, nums[2]=0
  非零？否，right++

步骤4：right=3, nums[3]=3
  非零？是！
  交换 nums[left] 和 nums[right]
  nums[1] 和 nums[3] 交换
  [1, 3, 0, 0, 12]
           ↑  ↑
         left right
  left++ → left=2

步骤5：right=4, nums[4]=12
  非零？是！
  交换 nums[left] 和 nums[right]
  nums[2] 和 nums[4] 交换
  [1, 3, 12, 0, 0]
              ↑  ↑
            left right
  left++ → left=3

遍历结束

最终结果：[1, 3, 12, 0, 0]
```

#### 特点

- 代码最简洁
- 操作次数最少（只交换必要的元素）
- **推荐使用这种方法**

## 三. 方法对比

| 方法 | 时间复杂度 | 空间复杂度 | 操作次数 | 推荐程度 |
|------|-----------|-----------|---------|---------|
| **冒泡排序法** | O(n²) | O(1) | 约 n²/2 次 | 不推荐 |
| **双指针法** | O(n) | O(1) | 最多 2n 次 | 推荐 |
| **交换法** | O(n) | O(1) | 最多 n 次 | **强烈推荐** |

### 复杂度分析

```
冒泡排序法：
  - 时间：O(n²) - 每个0都需要多次冒泡
  - 空间：O(1) - 原地操作
  - 交换次数：多

双指针法/交换法：
  - 时间：O(n) - 只需遍历一次
  - 空间：O(1) - 原地操作
  - 交换次数：少（只交换必要的元素）
```

## 五. 小贴士

### 1. 为什么必须原地操作？

```
面试官可能的考察点：
  - 不能使用额外的数组
  - 不能改变数组的总长度
  - 只能通过交换或覆盖来实现

常见的错误：
  - 使用 nums.remove(0) 和 nums.append(0)
    → remove 和 append 会改变列表长度，且操作复杂

正确的做法：
  - 使用双指针交换
  - 或者移动非零元素后填充零
```

### 2. 如何保持非零元素的相对顺序？

```
关键点：
  - 从左到右遍历，保证非零元素的相对位置不变
  - 不使用随机交换（如快速排序中的分区操作）

错误做法：
  - 类似快速排序的分区操作
  - 可能改变非零元素的相对顺序

正确做法：
  - 按顺序将非零元素移动到前面
  - 剩余位置填零
```

### 3. 进阶要求的优化

```
问题：如何减少操作次数？

方案1：交换法（推荐）
  - 只在需要时交换
  - 每个非零元素最多交换一次
  - 总操作次数：O(n)

方案2：记录位置，最后填充
  - 先记录所有非零元素的位置
  - 将非零元素移动到新位置
  - 剩余位置填零
  - 操作次数可能更少
```

## 六. 完整代码实现

### Python（交换法 - 最优解）

```python
from typing import List

class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        """
        移动零 - 交换法
        
        时间复杂度：O(n)
        空间复杂度：O(1)
        
        算法步骤：
            1. 使用双指针：left 和 right
            2. right 指针遍历数组
            3. 遇到非零元素时，与 left 指针交换
            4. left 指针之后的都是0
            
        Args:
            nums: 输入的整数数组
            
        Returns:
            None（原地修改数组）
        """
        n = len(nums)
        left = 0  # 左指针：指向下一个非零元素应该存放的位置
        
        for right in range(n):
            # 如果当前元素非零
            if nums[right] != 0:
                # 交换左右指针位置的元素
                nums[left], nums[right] = nums[right], nums[left]
                left += 1
```

### C++（交换法）

```cpp
#include <vector>

class Solution {
public:
    /**
     * 移动零 - 交换法
     * 
     * 时间复杂度：O(n)
     * 空间复杂度：O(1)
     */
    void moveZeroes(std::vector<int>& nums) {
        int n = nums.size();
        int left = 0;  // 左指针
        
        for (int right = 0; right < n; right++) {
            // 如果当前元素非零
            if (nums[right] != 0) {
                // 交换
                std::swap(nums[left], nums[right]);
                left++;
            }
        }
    }
};
```
