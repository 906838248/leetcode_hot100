# 项目规范

本技能确保所有代码开发都遵循项目的既定标准和规范。

## 何时调用

**在任何以下操作之前必须调用此技能：**
- 生成任何代码（任何编程语言）
- 修改任何代码（任何编程语言）
- 创建新文件或模块
- 完成任何开发任务
- 编写文档
- 代码审查或重构
- 添加注释或文档
- **所有涉及代码的操作**

## 语言要求

### 回复语言
- **必须使用中文回答**：所有与用户的交流、代码注释、文档说明必须使用中文
- 保持语言一致性，不要中英文混杂
- 代码注释也使用中文
- 例外情况：如果用户明确要求使用其他语言，或者引用的是英文技术文档/代码，则可以保留英文

## 核心原则

### 1. 代码组织与可维护性
- 保持代码组织良好且模块化
- 遵循一致的命名规范
- 适当分离关注点
- 编写自文档化的代码

### 2. 文件创建决策
- 自行判断代码要不要创建在新文件
- 仅在必要时创建新文件
- 适当重用现有模块
- 保持清晰的文件结构

### 3. 基于内容开发
- 确保生成的内容、代码、方案基于实际内容、代码，不要过分臆想
- 基于实际项目背景制定解决方案
- 避免没有证据的假设
- 参考代码库中的现有模式

## 代码文档标准

### 必需的注释格式

所有生成的函数和方法必须包含适当的文档注释。

#### Python 示例

```python

    """
    在屏幕上查找指定图片
    
    Args:
        image_name: 图片文件名

        timeout: 超时时间（秒）
        
    Returns:
        找到则返回(center_x, center_y)，否则返回None
    """
```

### 文档要求
- 所有公共函数和类都必须使用文档注释
- 包含参数描述和类型信息
- 记录返回值
- 清晰地解释目的和行为
- 保持语言一致性（中英文不要混杂）

## 最佳实践

1. **一致性**：遵循现有代码模式
2. **清晰性**：编写清晰易懂的代码
3. **模块化**：保持函数专注且单一职责
4. **多语言支持**：本规范适用于所有编程语言，包括但不限于：
   - Python
   - C++
   - Java
   - JavaScript/TypeScript
   - Go
   - Rust
   - 以及其他所有编程语言
