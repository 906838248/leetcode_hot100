# 12. 最小覆盖子串 - 详细解题分析

## 一. 题目

给定两个字符串 `s` 和 `t`，长度分别是 `m` 和 `n`，返回 `s` 中的最短窗口子串，使得该子串包含 `t` 中的每一个字符（包括重复字符）。如果没有这样的子串，返回空字符串 `""`。

测试用例保证答案唯一。

### 示例

**示例 1：**
```
输入: s = "ADOBECODEBANC", t = "ABC"
输出: "BANC"
解释: 
  最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。
```

**示例 2：**
```
输入: s = "a", t = "a"
输出: "a"
解释: 
  整个字符串 s 是最小覆盖子串。
```

**示例 3：**
```
输入: s = "a", t = "aa"
输出: ""
解释: 
  t 中两个字符 'a' 均应包含在 s 的子串中，
  因此没有符合条件的子字符串，返回空字符串。
```

### 提示

- `1 <= s.length, t.length <= 10^5`
- `s` 和 `t` 由英文字母组成

**进阶**：你能设计一个在 `O(m + n)` 时间内解决此问题的算法吗？

---

## 二. 解题思路

### 1. 问题分析

**目标**：在字符串 `s` 中找到最短的连续子串，这个子串包含字符串 `t` 中的所有字符

**关键约束**：
- 子串必须是连续的
- 需要包含 `t` 中每个字符至少一次
- 字符可以有重复（如 `t = "aa"` 表示需要至少两个 `'a'`）
- 如果找不到这样的子串，返回空字符串

**核心思想**：
- **滑动窗口**：使用双指针维护一个可变大小的窗口
- **哈希表**：记录需要的字符和窗口中的字符
- **计数匹配**：跟踪已满足条件的字符种类数

---

### 解法一：滑动窗口法（推荐）

#### 思路

使用滑动窗口算法，分为两个阶段：
1. **扩大窗口**：右指针向右移动，添加字符到窗口
2. **收缩窗口**：当窗口满足条件时，左指针向右移动找最小窗口

```python
from typing import List
from collections import defaultdict

class Solution:
    def minWindow(self, s: str, t: str) -> str:
        """
        最小覆盖子串 - 滑动窗口法

        时间复杂度：O(m + n)
        空间复杂度：O(m + n)

        算法步骤：
            1. 用哈希表记录 t 中每个字符需要的数量
            2. 用双指针维护滑动窗口
            3. 扩大右指针，添加字符到窗口
            4. 当窗口满足条件时，收缩左指针找最小窗口
            5. 重复直到遍历完整个字符串
        """
        if len(t) > len(s):
            return ""
        
        need_count = defaultdict(int)
        window = defaultdict(int)
        
        for char in t:
            need_count[char] += 1
        
        required = len(need_count)
        formed = 0
        left = 0
        min_length = float('inf')
        min_window = ""
        
        for right, char in enumerate(s):
            window[char] += 1
            
            if char in need_count and window[char] == need_count[char]:
                formed += 1
            
            while formed == required:
                current_length = right - left + 1
                
                if current_length < min_length:
                    min_length = current_length
                    min_window = s[left:right+1]
                
                left_char = s[left]
                window[left_char] -= 1
                
                if left_char in need_count and window[left_char] < need_count[left_char]:
                    formed -= 1
                
                left += 1
        
        return min_window
```

#### 为什么滑动窗口法可行？

```
核心思想：

  1. 可变大小的窗口：
     - 右指针扩展窗口，添加字符
     - 左指针收缩窗口，移除字符
     - 窗口大小根据条件动态调整

  2. 计数匹配机制：
     - need_count：记录 t 中每个字符需要的数量
     - window：记录窗口中每个字符的数量
     - formed：记录已满足条件的字符种类数

  3. 寻找最小窗口：
     - 当窗口满足条件时，记录当前长度
     - 尝试收缩左指针，看是否能找到更短的窗口
     - 直到窗口不再满足条件

  4. 时间复杂度分析：
     - 每个字符最多被右指针访问一次
     - 每个字符最多被左指针访问一次
     - 总共 O(m + n) 次操作
```

### 解法二：笨办法（暴力枚举）

#### 思路

枚举所有可能的子串，检查是否包含 t 中的所有字符。

```python
from typing import List

class Solution:
    def minWindow(self, s: str, t: str) -> str:
        """
        最小覆盖子串 - 暴力枚举法

        时间复杂度：O(m² * n)
        空间复杂度：O(m + n)
        """
        if len(t) > len(s):
            return ""
        
        result = ""
        min_length = float('inf')
        
        for i in range(len(s)):
            for j in range(i + 1, len(s) + 1):
                substring = s[i:j]
                if self._contains_all(substring, t):
                    if j - i < min_length:
                        min_length = j - i
                        result = substring
        
        return result
    
    def _contains_all(self, s: str, t: str) -> bool:
        from collections import Counter
        s_count = Counter(s)
        t_count = Counter(t)
        for char, count in t_count.items():
            if s_count[char] < count:
                return False
        return True
```

#### 复杂度分析

```
暴力枚举法：
  时间复杂度：O(m² * n)
    - 外层循环 m 次（起始位置）
    - 内层循环 m 次（结束位置）
    - 每次检查 O(n)
    - 总共 O(m² * n)

  空间复杂度：O(m + n)

滑动窗口法：
  时间复杂度：O(m + n)
    - 每个字符最多被左右指针各访问一次

  空间复杂度：O(m + n)
```

### C++ 实现

```cpp
#include <unordered_map>
#include <string>
#include <iostream>
using namespace std;

class Solution {
public:
    string minWindow(string s, string t) {
        if (t.length() > s.length()) {
            return "";
        }
        
        unordered_map<char, int> need_count;
        unordered_map<char, int> window;
        
        for (char c : t) {
            need_count[c]++;
        }
        
        int required = need_count.size();
        int formed = 0;
        int left = 0;
        int min_length = INT_MAX;
        string min_window = "";
        
        for (int right = 0; right < s.length(); right++) {
            char c = s[right];
            window[c]++;
            
            if (need_count.find(c) != need_count.end() && window[c] == need_count[c]) {
                formed++;
            }
            
            while (formed == required) {
                int current_length = right - left + 1;
                
                if (current_length < min_length) {
                    min_length = current_length;
                    min_window = s.substr(left, current_length);
                }
                
                char left_char = s[left];
                window[left_char]--;
                
                if (need_count.find(left_char) != need_count.end() && window[left_char] < need_count[left_char]) {
                    formed--;
                }
                
                left++;
            }
        }
        
        return min_window;
    }
};
```

## 五. 滑动窗口的深入理解

### 1. 什么是滑动窗口？

**滑动窗口**是一种双指针技术，通过两个指针维护一个窗口（区间），窗口可以动态扩展和收缩。

```
普通滑动窗口：窗口大小固定
可变滑动窗口：窗口大小可变（本问题使用）

示例：
  字符串 s = "ADOBECODEBANC"
  t = "ABC"

  窗口变化过程：
    [A]DO...              长度 = 1
    [A, D]OB...           长度 = 2
    [A, D, O, B]E...      长度 = 4
    ...
    [B, A, N, C]OD...     长度 = 4 (最小覆盖子串)
```

### 2. 为什么叫"最小覆盖子串"？

**覆盖**：窗口中的字符覆盖了 t 中所有字符的需求

**最小**：在所有满足条件的窗口中，长度最小的那个

### 3. 滑动窗口的三个关键点

```python
# 1. 扩大窗口（右指针移动）
for right, char in enumerate(s):
    window[char] += 1

# 2. 检查条件（是否满足要求）
if formed == required:
    # 满足条件

# 3. 收缩窗口（左指针移动）
while formed == required:
    # 记录答案
    left_char = s[left]
    window[left_char] -= 1
    left += 1
```

### 4. 直观理解

**类比**：想象你在找一张包含所有必需物品的最小袋子。

- **t**：需要的物品清单（`'A'`、`'B'`、`'C'` 各一个）
- **s**：一排物品
- **滑动窗口**：你拿着一个袋子，从左到右扫过物品
- **目标**：找到最短的袋子，包含清单上的所有物品

| 解法 | 时间复杂度 | 空间复杂度 | 说明 |

| 滑动窗口法 | O(m + n) | O(m + n) | 推荐解法 |

**关键点：**
1. 理解哈希表如何记录字符需求和窗口状态
2. 掌握 `formed` 计数机制：跟踪已满足条件的字符种类数
3. 理解窗口的扩展和收缩过程
4. 每个字符最多被左右指针各访问一次，保证 O(m + n) 时间复杂度

- 子字符串匹配问题
- 字符覆盖问题
