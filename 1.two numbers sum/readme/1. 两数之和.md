# 一. 题目


给定一个整数数组 `nums`和一个整数目标值 `target`，请你在该数组中找出 **和为目标值**​ target的那 **两个**​ 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

---

## 1. 示例

**示例 1：**

```
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。
```

**示例 2：**

```
输入：nums = [3,2,4], target = 6
输出：[1,2]
```

**示例 3：**

```
输入：nums = [3,3], target = 6
输出：[0,1]
```

---

## 2. 提示

- `2 <= nums.length <= 104`
    
- `-109 <= nums[i] <= 109`
    
- `-109 <= target <= 109`
    
- 只会存在一个有效答案

# 二. 解题思路


### 1. 问题分析

**目标**：在数组中找到两个数，使它们的和等于目标值 `target`

**核心思想**：对于每个元素 `num`，我们需要找到 `target - num` 是否存在于数组中

---

## 解法一：暴力枚举（Brute Force）

### 思路
两层嵌套循环，枚举所有可能的数对。

```python
def two_sum(nums, target):
    """
    暴力枚举方法
    
    时间复杂度：O(n²)
    空间复杂度：O(1)
    """
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []
```

### 图解过程
```
数组：[2, 7, 11, 15]，目标值：9

第1次遍历：
  i=0, 检查 nums[0]=2
    j=1: 2+7=9 ✓ 找到答案！返回 [0, 1]
```

### 优缺点
- ✅ 简单直接，易于理解
- ❌ 时间复杂度高，不适合大数据集

---

##  解法二：哈希表（推荐！）

### 核心思想
利用哈希表存储已遍历过的元素及其索引，**一次遍历**就能找到答案。

```python
def two_sum(nums, target):
    """
    哈希表方法（推荐）
    
    时间复杂度：O(n)
    空间复杂度：O(n)
    """
    seen = {}  # 创建哈希表：{数值: 索引}
    
    for i, num in enumerate(nums):
        complement = target - num  # 计算需要的另一个数
        if complement in seen:       # 检查是否在哈希表中
            return [seen[complement], i]  # 找到了！
        seen[num] = i              # 没找到，存入哈希表
    return []
```

### 图解过程
```
数组：[2, 7, 11, 15]，目标值：9

步骤1：i=0, num=2
  complement = 9 - 2 = 7
  7 in seen? No
  seen = {2: 0}

步骤2：i=1, num=7
  complement = 9 - 7 = 2
  2 in seen? Yes! seen[2]=0
  ✓ 找到答案：[0, 1]
```

### 为什么哈希表快？
- **查找速度快**：O(1) 时间复杂度
- **只需一次遍历**：从 O(n²) 降低到 O(n)

---

## 解法三：双指针（已排序数组）

### 核心思想
如果数组已排序，可以用两个指针从两端向中间逼近。

```python
def two_sum_sorted(sorted_nums, target):
    """
    双指针方法（适用于已排序数组）
    
    时间复杂度：O(n log n)（排序）+ O(n)（双指针）= O(n log n)
    空间复杂度：O(1)
    """
    left = 0
    right = len(sorted_nums) - 1
    
    while left < right:
        current_sum = sorted_nums[left] + sorted_nums[right]
        
        if current_sum < target:
            left += 1      # 和太小，左指针右移
        elif current_sum > target:
            right -= 1    # 和太大，右指针左移
        else:
            return [left, right]  # 找到答案
    return []
```

### 图解过程
```
已排序数组：[2, 7, 11, 15]，目标值：9

步骤1：left=0, right=3
  sum = 2 + 15 = 17 > 9
  right -= 1 → right=2

步骤2：left=0, right=2
  sum = 2 + 11 = 13 > 9
  right -= 1 → right=1

步骤3：left=0, right=1
  sum = 2 + 7 = 9 = target
  ✓ 找到答案：[0, 1]
```

---

# 三. 方法对比

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| **暴力枚举** | O(n²) | O(1) | 小数据集，面试时快速实现 |
| **哈希表** ⭐ | O(n) | O(n) | **一般情况下的最优解** |
| **双指针** | O(n log n) | O(1) | 数组已排序时 |

---

# 四. 小贴士

1. **首选哈希表**：时间复杂度最优
2. **考虑边界情况**：
   - 空数组
   - 没有符合条件的数对
   - 数组中有重复元素
3. **哈希表方法的优势**：
   - 一次遍历解决问题
   - 正确处理重复元素
   - 不需要修改原数组（无需排序）
