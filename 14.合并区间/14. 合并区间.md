# 14. 合并区间 - 详细解题分析

## 一. 题目

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [start_i, end_i]`。请你合并所有重叠的区间，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。

### 示例

**示例 1：**
```
输入: intervals = [[1,3],[2,6],[8,10],[15,18]]
输出: [[1,6],[8,10],[15,18]]
解释: 区间 [1,3] 和 [2,6] 重叠, 将它们合并为 [1,6]。
```

**示例 2：**
```
输入: intervals = [[1,4],[4,5]]
输出: [[1,5]]
解释: 区间 [1,4] 和 [4,5] 可被视为重叠区间。
```

**示例 3：**
```
输入: intervals = [[4,7],[1,4]]
输出: [[1,7]]
解释: 区间 [1,4] 和 [4,7] 可被视为重叠区间。
```

### 提示

- `1 <= intervals.length <= 10^4`
- `intervals[i].length == 2`
- `0 <= start_i <= end_i <= 10^4`

---

## 二. 解题思路

### 1. 问题分析

**目标**：合并所有重叠的区间，返回不重叠的区间数组

**关键约束**：
- 区间是连续的 [start, end]
- 需要合并所有有重叠的区间
- 相邻的区间也算重叠（如 [1,4] 和 [4,5]）
- 返回的区间必须覆盖所有输入区间

**核心思想**：
- **排序**：先将区间按起始位置排序
- **贪心合并**：遍历排序后的区间，合并重叠的区间
- **关键洞察**：排序后只需要比较相邻区间，大大降低复杂度

---

## 三. 算法详解

### 1. 排序 - 预处理

为什么需要排序？

```
未排序的区间：
  [[1,3], [8,10], [2,6], [15,18]]
  
排序后的区间：
  [[1,3], [2,6], [8,10], [15,18]]

排序的作用：
  - 相邻区间现在在数组中也是相邻的
  - 只需要比较相邻区间是否重叠
  - 不需要 O(n²) 的两两比较
```

### 2. 合并判断

两个区间 `[a, b]` 和 `[c, d]` 是否重叠？

```
重叠的两种情况：

情况1：区间1包含区间2的开始
  [a, b]  和  [c, d]
    a  b
      c  d
  条件：c <= b

情况2：区间2包含区间1的开始
  [a, b]  和  [c, d]
      a  b
    c  d
  条件：a <= d

统一条件：如果 a <= d 且 c <= b，两个区间重叠

但由于我们排序后 a <= c，所以简化为：
  如果 c <= b，两个区间重叠
```

### 3. 合并操作

当两个区间重叠时，合并为 `[a, max(b, d)]`：

```
重叠区间：
  [1, 3]  和  [2, 6]
    a  b      c  d
    a=1, b=3, c=2, d=6
    
合并条件：c <= b → 2 <= 3 ✓

合并结果：
  [a, max(b, d)]
  = [1, max(3, 6)]
  = [1, 6]
```

## 四. 算法实现

### 1. 核心代码

```python
from typing import List

class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        """
        合并区间

        时间复杂度：O(n log n)
        空间复杂度：O(1) （不计输出空间）

        算法步骤：
            1. 按区间的起始位置排序
            2. 遍历区间，合并重叠的区间
            3. 判断重叠：当前区间起始 ≤ 上一个合并区间结束
            4. 合并：更新合并区间的结束位置为较大值
        """
        if len(intervals) <= 1:
            return intervals

        intervals.sort(key=lambda x: x[0])

        merged = [intervals[0]]

        for i in range(1, len(intervals)):
            current = intervals[i]
            last_merged = merged[-1]

            if current[0] <= last_merged[1]:
                last_merged[1] = max(last_merged[1], current[1])
            else:
                merged.append(current)

        return merged
```

### 2. 代码逐行解析

```python
# 第一步：边界检查
if len(intervals) <= 1:
    return intervals

# 第二步：按起始位置排序
intervals.sort(key=lambda x: x[0])

# 第三步：初始化合并结果
merged = [intervals[0]]

# 第四步：遍历并合并
for i in range(1, len(intervals)):
    current = intervals[i]          # 当前区间
    last_merged = merged[-1]         # 上一个合并的区间
    
    # 判断是否重叠
    if current[0] <= last_merged[1]:
        # 重叠：更新结束位置为较大值
        last_merged[1] = max(last_merged[1], current[1])
    else:
        # 不重叠：直接添加
        merged.append(current)

return merged
```

## 七. 复杂度分析

### 时间复杂度：O(n log n)

```
1. 排序：O(n log n)
2. 遍历合并：O(n)
3. 总计：O(n log n) + O(n) = O(n log n)

为什么排序是必须的？
  - 如果不排序，需要 O(n²) 的两两比较
  - 排序后只需要比较相邻区间
  - 将复杂度从 O(n²) 降到 O(n log n)
```

### 空间复杂度：O(1) 或 O(n)

```
方案1：不修改输入，创建新列表
  merged = [intervals[0]]  ← O(n) 空间
  
方案2：原地修改（如果允许）
  intervals.sort()  ← O(1) 空间
  原地更新区间
  
通常计为：O(1) （不计输出空间）
```

### 复杂度对比

```
┌─────────────┬──────────────┬──────────────┐
│   方案      │  时间复杂度   │  空间复杂度  │
├─────────────┼──────────────┼──────────────┤
│  排序+合并  │  O(n log n)  │    O(n)      │
├─────────────┼──────────────┼──────────────┤
│  暴力比较   │   O(n²)      │    O(1)      │
└─────────────┴──────────────┴──────────────┘

推荐方案：排序+合并（平衡时间和空间）
```

## 八. 为什么排序后只需要比较相邻区间？

这是一个关键的数学洞察：

### 定理

如果区间按起始位置排序，那么区间 i 和区间 j（i < j）重叠，
当且仅当区间 i 和区间 i+1, i+2, ..., j-1 都重叠。

### 直观理解

```
排序后的区间：
  [1, 3], [2, 6], [4, 8], [7, 10]

假设 [1,3] 和 [7,10] 重叠吗？
  [1, 3]───────
              [4, 8]────────
                         [7, 10]
  
  显然不重叠！

为什么？
  - [1,3] 的结束位置是 3
  - [2,6] 覆盖了 3，所以和 [1,3] 重叠
  - [4,8] 也覆盖了 3
  - 但 [7,10] 的起始位置是 7 > 3，不在 [1,3] 范围内
  - 更关键的是：[4,8] 和 [7,10] 之间没有其他区间
  
  如果 [1,3] 和 [7,10] 重叠，那么 [7,10] 必须和 [4,8] 重叠
  
  所以我们只需要检查相邻区间！
```

### 形式化证明

```
设排序后的区间为 I1, I2, ..., In
其中 Ii = [ai, bi]，且 a1 <= a2 <= ... <= an

如果 Ik 和 Im 重叠（k < m），则：
  ak <= bm  （因为 Ik 的起始位置）
  且 bm <= bm  （因为区间 m 的结束位置）
  
关键观察：
  对于任意 i 在 k 和 m 之间，有：
  ak <= ai <= bi （因为排序）
  bi >= ak       （因为 ai >= a1 >= ak）
  
  所以如果 Ik 和 Im 重叠，那么 Ik 和 Ii 也重叠！
  
结论：只需要检查相邻区间
```

### C++实现

```cpp
#include <vector>
#include <algorithm>
using namespace std;

class Solution {
public:
    vector<vector<int>> merge(vector<vector<int>>& intervals) {
        if (intervals.size() <= 1) {
            return intervals;
        }

        sort(intervals.begin(), intervals.end(),
             [](const vector<int>& a, const vector<int>& b) {
                 return a[0] < b[0];
             });

        vector<vector<int>> merged;
        merged.push_back(intervals[0]);

        for (int i = 1; i < intervals.size(); i++) {
            vector<int>& current = intervals[i];
            vector<int>& last_merged = merged.back();

            if (current[0] <= last_merged[1]) {
                last_merged[1] = max(last_merged[1], current[1]);
            } else {
                merged.push_back(current);
            }
        }

        return merged;
    }
};
```

## 十一. 深入理解

### 1. 为什么从排序开始？

**核心洞察**：排序将二维问题降为一维问题

```
未排序：
  区间在任意位置，需要两两比较
  [1,3] 可能和 [8,10] 比较
  [2,6] 可能和 [15,18] 比较
  总共需要 O(n²) 次比较

排序后：
  区间按起始位置排列
  [1,3] 只需要和 [2,6] 比较（相邻）
  [2,6] 只需要和 [8,10] 比较（相邻）
  总共只需要 O(n) 次比较

关键：排序是 O(n log n)，比较是 O(n)
      总复杂度：O(n log n) + O(n) = O(n log n)
```

### 2. 贪心策略

本题使用贪心策略：**局部最优导致全局最优**

```
贪心选择：
  每一步都尽可能合并区间
  - 如果当前区间和上一个区间重叠，合并
  - 否则，开始新的区间

为什么贪心有效？
  - 排序后，区间是按时间顺序排列的
  - 合并操作是确定性的（没有多种选择）
  - 局部最优（合并相邻重叠区间）= 全局最优（合并所有重叠区间）

类比：
  想象你在整理书架上的书本区间
  - 按位置从左到右排列
  - 如果两本书有重叠（放不下），合并成一堆
  - 继续处理下一本
  - 最终得到最小的不重叠区间堆数
```

### 3. 边界情况处理

**边界重叠的定义**：
```
[1,4] 和 [4,5] 算重叠吗？
  - 从数学角度：[1,4] 和 [4,5] 在点 4 相交
  - 从区间覆盖角度：两个区间一起覆盖 [1,5]
  - 所以算作重叠 ✓

代码实现：
  if current[0] <= last_merged[1]:
      # 重叠（包括边界）
```

## 十二. 总结

### 算法对比

| 解法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 排序+合并 | O(n log n) | O(n) | **推荐**，最优解 |
| 暴力比较 | O(n²) | O(1) | 简单但效率低 |

### 关键点总结

1. **排序是核心**：按起始位置排序，将二维问题降为一维
2. **相邻比较**：排序后只需要比较相邻区间
3. **贪心策略**：局部最优导致全局最优
4. **边界处理**：`current[0] <= last_merged[1]` 包括边界重叠

### 算法流程

```
1. 边界检查：区间数量 <= 1，直接返回
2. 排序：按起始位置升序排序
3. 初始化：merged = [第一个区间]
4. 遍历合并：
   - 如果当前区间起始 <= 上一个区间结束 → 合并
   - 否则 → 添加新区间
5. 返回合并结果
```
