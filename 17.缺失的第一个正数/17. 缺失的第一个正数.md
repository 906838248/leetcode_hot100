# 17. 缺失的第一个正数 - 详细解题分析

## 一. 题目

给你一个未排序的整数数组 `nums`，请你找出其中没有出现的最小正整数。

请你实现时间复杂度为 O(n) 并且只使用常数级别额外空间的解决方案。

### 示例

**示例 1：**
```
输入: nums = [1,2,0]
输出: 3
解释: 范围 [1,2] 中的数字都在数组中。
```

**示例 2：**
```
输入: nums = [3,4,-1,1]
输出: 2
解释: 1 在数组中，但 2 没有。
```

**示例 3：**
```
输入: nums = [7,8,9,11,12]
输出: 1
解释: 最小的正数 1 没有出现。
```

### 提示

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`

**进阶**：
- 你可以实现时间复杂度为 O(n) 并使用常数级别额外空间的解决方案吗？

## 二. 解题思路

### 1. 问题分析

**目标**：找出未排序数组中没有出现的最小正整数

**关键约束**：
- 时间复杂度：O(n)
- 空间复杂度：O(1)（常数级别）
- 数组未排序，可能包含负数和重复元素

**核心洞察**：
- 对于长度为 n 的数组，答案一定在 [1, n+1] 范围内
- 可以利用数组的索引来标记数字是否出现
- 将数字 k 放到索引 k-1 的位置

### 2. 关键洞察

```
对于数组 nums = [3,4,-1,1]：

答案一定在 [1, n+1] = [1, 5] 范围内：
  - 可能的答案：1, 2, 3, 4, 5
  - 如果 1,2,3,4 都出现，答案是 5
  - 否则是第一个没出现的正数

核心思想：
  - 将数字 num 放到索引 num-1 的位置
  - 例如：数字 3 应该放到索引 2
  - 如果索引 i 的位置不是 i+1，说明数字 i+1 缺失
```

## 三. 解法一：暴力法（排序）

### 1. 算法思想

先对数组排序，然后遍历找出第一个缺失的正数。

### 2. 代码实现

#### Python实现

```python
from typing import List

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        """
        缺失的第一个正数 - 排序法

        时间复杂度：O(n log n)
        空间复杂度：O(1) 或 O(n)（取决于排序算法）

        说明：不满足进阶要求的 O(n) 时间复杂度
        """
        nums.sort()
        result = 1

        for num in nums:
            if num == result:
                result += 1
            elif num > result:
                break

        return result
```

### 3. 复杂度分析

```
时间复杂度：O(n log n)
  - 排序操作：O(n log n)
  - 遍历操作：O(n)

空间复杂度：O(1) 或 O(n)
  - 如果是原地排序：O(1)
  - 如果不是原地排序：O(n)

结论：不满足进阶要求的 O(n) 时间复杂度
```

## 四. 解法二：哈希表法

### 1. 算法思想

使用哈希表记录数组中出现的所有正整数，然后从 1 开始检查。

### 2. 代码实现

#### Python实现

```python
from typing import List

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        """
        缺失的第一个正数 - 哈希表法

        时间复杂度：O(n)
        空间复杂度：O(n)

        说明：不满足进阶要求的 O(1) 空间复杂度
        """
        num_set = set(nums)
        result = 1

        while result in num_set:
            result += 1

        return result
```

### 3. 复杂度分析

```
时间复杂度：O(n)
  - 创建哈希表：O(n)
  - 循环检查：最坏 O(n)

空间复杂度：O(n)
  - 需要额外的哈希表存储所有元素

结论：不满足进阶要求的 O(1) 空间复杂度
```

## 五. 解法三：原地哈希法【推荐】

### 1. 算法思想

利用数组的索引来标记数字是否出现：
- 将数字 k 放到索引 k-1 的位置
- 遍历数组，将符合条件的数字放到正确位置
- 最后遍历数组，第一个位置 i 不是 i+1 的就是答案

### 2. 算法步骤

```
1. 遍历数组（i 从 0 到 n-1）：
   a. 当 nums[i] 在 [1, n] 范围内且 nums[nums[i]-1] != nums[i] 时：
      - 交换 nums[i] 和 nums[nums[i]-1]
      - 继续处理当前位置（因为交换后可能是新的有效数字）
   b. 否则，i++
2. 再次遍历数组：
   a. 如果 nums[i] != i+1，返回 i+1
3. 如果所有位置都正确，返回 n+1
```

### 3. 代码实现

#### Python实现

```python
from typing import List

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        """
        缺失的第一个正数 - 原地哈希法

        时间复杂度：O(n)
        空间复杂度：O(1)

        算法思想：
          1. 将数字 k 放到索引 k-1 的位置
          2. 遍历数组，将符合条件的数字放到正确位置
          3. 再次遍历，找到第一个位置 i 不是 i+1 的
        """
        n = len(nums)

        for i in range(n):
            while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
                correct_idx = nums[i] - 1
                nums[i], nums[correct_idx] = nums[correct_idx], nums[i]

        for i in range(n):
            if nums[i] != i + 1:
                return i + 1

        return n + 1
```

#### C++实现

```cpp
#include <vector>
using namespace std;

class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        int n = nums.size();

        for (int i = 0; i < n; i++) {
            while (1 <= nums[i] && nums[i] <= n && nums[nums[i] - 1] != nums[i]) {
                swap(nums[i], nums[nums[i] - 1]);
            }
        }

        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }

        return n + 1;
    }
};
```

```
  - 总共 O(n) 操作

空间复杂度：O(1)
  - 只使用了常数个额外变量（i, n, correct_idx）
  - 没有使用额外的数据结构
  - 满足进阶要求
```

## 六. 原地哈希法的深入理解

### 1. 为什么答案是 [1, n+1]？

```
对于长度为 n 的数组：
  - 最多可以表示 n 个连续的正整数 [1, n]
  - 如果所有 [1, n] 都出现，答案是 n+1
  - 否则，答案是第一个没出现的正整数

例子：
  nums = [1, 2, 3], n = 3
    答案在 [1, 4] 范围内
    [1, 2, 3] 都出现 → 答案是 4

  nums = [1, 3], n = 2
    答案在 [1, 3] 范围内
    1 出现，2 没出现 → 答案是 2
```

### 2. 为什么将数字 k 放到索引 k-1？

```
映射关系：
  数字 1 → 索引 0
  数字 2 → 索引 1
  数字 3 → 索引 2
  ...
  数字 k → 索引 k-1

验证：
  如果数字都在正确位置：[1, 2, 3, 4]
                    → 索引 [0, 1, 2, 3]
  nums[i] == i+1 对所有 i 成立
```

### 3. 为什么使用 while 循环？

```
在交换后，新位置的数字可能也需要移动：

例子：
  nums = [3, 4, -1, 1]
  
  i=1, nums[1]=4
    交换：nums[1] 和 nums[3]
    nums = [-1, 1, 3, 4]
  
  i=1, nums[1]=1（新数字）
    交换：nums[1] 和 nums[0]
    nums = [1, -1, 3, 4]
  
  i=1, nums[1]=-1（新数字）
    不满足条件，跳过

使用 while 确保所有数字都被放到正确位置
```

### 4. 为什么条件是 `1 <= nums[i] <= n`？

```
只处理在 [1, n] 范围内的数字：
  - 小于 1 的数字（0, 负数）：忽略
  - 大于 n 的数字：不可能是答案（答案最大是 n+1）

例子：
  nums = [7, 8, 9, 11, 12], n = 5
    所有数字都大于 5
    不做任何交换
    第一遍遍历后直接返回 1
```

```python
    def firstMissingPositive(self, nums: List[int]) -> int:

        缺失的第一个正数 - 原地哈希法
```

### 核心算法

```
原地哈希法：
  1. 将数字 k 放到索引 k-1 的位置
  2. 遍历数组，将符合条件的数字放到正确位置
  3. 再次遍历，找到第一个位置 i 不是 i+1 的
```

### 关键点总结

1. **答案范围**：答案一定在 [1, n+1] 范围内
2. **索引映射**：将数字 k 放到索引 k-1 的位置
3. **原地操作**：不需要额外的数据结构
4. **while 循环**：确保所有数字都被放到正确位置

### 最终建议

```
对于缺失的第一个正数这个问题：

  ✅ 使用原地哈希法（O(n)时间，O(1)空间）
  ✅ 利用数组索引代替哈希表
  ✅ 使用 while 循环确保正确性
  ✅ 代码简洁高效
  
  需要理解的要点：
  ✅ 为什么答案是 [1, n+1]
  ✅ 为什么使用 while 循环
  ✅ 如何利用数组索引
  ✅ 如何处理各种边界情况
  
  需要避免的方法：
  ❌ 排序法：O(n log n) 时间复杂度
  ❌ 哈希表法：O(n) 空间复杂度
```
