

# 7. 接雨水 - 详细解题分析

## 一. 题目

给定 `n` 个非负整数表示每个宽度为 `1` 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

### 示例

**示例 1：**
```
输入: height = [0,1,0,2,1,0,1,3,2,1,2,1]
输出: 6
解释: 上面是由数组 [0,1,0,2,1,0,1,3,2,1,2,1] 表示的高度图，
     在这种情况下，可以接 6 个单位的雨水（蓝色部分表示雨水）。
     

```
![[leetcode_hot/双指针/PixPin_2026-04-09_15-05-22.png]]
**示例 2：**
```
输入: height = [4,2,0,3,2,5]
输出: 9
```

### 提示

- `n == height.length`
- `1 <= n <= 2 * 10^4`
- `0 <= height[i] <= 10^5`

## 二. 解题思路

### 1. 问题分析

**目标**：计算能接住的雨水总量

**关键约束**：
- 每个柱子宽度为 1
- 雨水只能存在于**两个较高柱子之间**
- 每个位置能接的雨水 = `min(左边最高柱子, 右边最高柱子) - 当前柱子高度`

**核心思想**：
- **暴力法**：对每个位置，计算左右最高柱子
- **备忘录优化**：预计算左右最高柱子
- **双指针优化**：左右指针从两端向中间逼近

---

### 解法一：暴力法（容易理解但低效）

#### 思路

对每个位置，分别找到它左边和右边最高的柱子，然后计算该位置能接的雨水。

```python
def trap(height: list[int]) -> int:
    """
    接雨水 - 暴力法
    
    时间复杂度：O(n²)
    空间复杂度：O(1)
    
    说明：
        - 对每个位置，遍历左边找最高，遍历右边找最高
        - 代码简单但效率低
    """
    n = len(height)
    water = 0
    
    for i in range(1, n - 1):
        # 找左边最高的柱子
        left_max = 0
        for j in range(i):
            left_max = max(left_max, height[j])
        
        # 找右边最高的柱子
        right_max = 0
        for j in range(i + 1, n):
            right_max = max(right_max, height[j])
        
        # 当前柱子能接的雨水
        water += min(left_max, right_max) - height[i]
    
    return water
```

#### 图解过程

```
数组：[0,1,0,2,1,0,1,3,2,1,2,1]

分析每个位置（跳过首尾）：

位置0：无法接水（边界）
位置1：左边最高=0，右边最高=3 → min(0,3)-1 = -1 → 不接水
位置2：左边最高=1，右边最高=3 → min(1,3)-0 = 1 → 接1单位水
位置3：左边最高=1，右边最高=3 → min(1,3)-2 = -1 → 不接水
位置4：左边最高=2，右边最高=3 → min(2,3)-1 = 1 → 接1单位水
位置5：左边最高=2，右边最高=3 → min(2,3)-0 = 2 → 接2单位水
位置6：左边最高=2，右边最高=3 → min(2,3)-1 = 1 → 接1单位水
位置7：无法接水（最高柱子）
位置8：左边最高=3，右边最高=2 → min(3,2)-2 = 0 → 不接水
位置9：左边最高=3，右边最高=2 → min(3,2)-1 = 0 → 不接水
位置10：左边最高=3，右边最高=1 → min(3,1)-2 = -1 → 不接水
位置11：无法接水（边界）

总水量：1 + 1 + 2 + 1 = 6 单位
```

#### 优缺点

- 优点：代码简单，容易理解
- 缺点：**时间复杂度 O(n²)**，对于大数组效率低

### 解法二：备忘录优化法（推荐）

#### 思路

预计算每个位置左边和右边的最高柱子，避免重复计算。

```python
def trap(height: list[int]) -> int:
    """
    接雨水 - 备忘录优化法（推荐）
    
    时间复杂度：O(n)
    空间复杂度：O(n)
    
    说明：
        - 预先计算 left_max 和 right_max 数组
        - 每个位置只需 O(1) 计算
        - 牺牲空间换时间
    """
    n = len(height)
    
    if n == 0:
        return 0
    
    # 预计算左边最高柱子
    left_max = [0] * n
    left_max[0] = height[0]
    for i in range(1, n):
        left_max[i] = max(left_max[i - 1], height[i])
    
    # 预计算右边最高柱子
    right_max = [0] * n
    right_max[n - 1] = height[n - 1]
    for i in range(n - 2, -1, -1):
        right_max[i] = max(right_max[i + 1], height[i])
    
    # 计算每个位置能接的雨水
    water = 0
    for i in range(1, n - 1):
        water += min(left_max[i], right_max[i]) - height[i]
    
    return water
```

#### 为什么备忘录优化法高效？

```
暴力法：
  - 对每个位置，都要重新遍历左边和右边
  - 总操作次数：n × n = O(n²)
  - 大量重复计算

备忘录法：
  - 预计算只需遍历两边：2n = O(n)
  - 计算雨水再遍历一遍：n = O(n)
  - 总操作次数：O(n) + O(n) + O(n) = O(n)
  - 无重复计算

性能提升：n 倍！
```

### 解法三：双指针法（最优！）

#### 思路

左右指针从两端向中间逼近，只关心每个位置的**有效边界**。

```python
def trap(height: list[int]) -> int:
    """
    接雨水 - 双指针法（最优解）
    
    时间复杂度：O(n)
    空间复杂度：O(1)
    
    说明：
        - 左右指针从两端向中间逼近
        - 每次只处理较小的一边
        - 无需额外空间
    """
    n = len(height)
    
    if n == 0:
        return 0
    
    left = 0
    right = n - 1
    left_max = 0
    right_max = 0
    water = 0
    
    while left < right:
        # 比较左右两边的柱子高度
        if height[left] < height[right]:
            # 左边较低，处理左边
            if height[left] >= left_max:
                # 如果当前高度 >= left_max，更新边界
                left_max = height[left]
            else:
                # 当前高度 < left_max，可以接雨水
                water += left_max - height[left]
            left += 1
        else:
            # 右边较低或相等，处理右边
            if height[right] >= right_max:
                # 如果当前高度 >= right_max，更新边界
                right_max = height[right]
            else:
                # 当前高度 < right_max，可以接雨水
                water += right_max - height[right]
            right -= 1
    
    return water
```

#### 图解过程

```
数组：[0,1,0,2,1,0,1,3,2,1,2,1]

初始状态：
  left = 0, right = 11
  left_max = 0, right_max = 0
  water = 0
  
  [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
   L                                   R

步骤1：height[left]=0 < height[right]=1，处理左边
  height[left]=0 >= left_max=0 → left_max=0
  left++ → left=1
  
  [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
      L                                 R

步骤2：height[left]=1 < height[right]=1，处理左边
  height[left]=1 >= left_max=0 → left_max=1
  left++ → left=2
  
  [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
          L                               R

步骤3：height[left]=0 < height[right]=1，处理左边
  height[left]=0 < left_max=1 → 接雨水
  water += 1 - 0 = 1
  left++ → left=3
  
  [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
              L                            R

步骤4：height[left]=2 < height[right]=1？否，处理右边
  height[right]=1 >= right_max=0 → right_max=1
  right-- → right=10
  
  [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
              L                          R

步骤5：height[left]=2 > height[right]=2，处理右边
  height[right]=2 >= right_max=1 → right_max=2
  right-- → right=9
  
  [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
              L                       R

...继续处理，最终得到 water = 6
```

#### 为什么双指针法可行？

```
核心思想：
  
  1. 某个位置能接多少水，由「较小的那边边界」决定
     - 如果左边边界 < 右边边界，水量由左边决定
     - 如果右边边界 < 左边边界，水量由右边决定

  2. 指针移动策略：
     - 总是移动「较小的一边」
     - 因为较小边决定了水量，必须先处理它

  3. 证明正确性：
     - 左边指针只依赖 left_max（已知）
     - 右边指针只依赖 right_max（已知）
     - 两边互相独立，可以分别处理
```

  - 时间：O(n) - 单次遍历
  - 空间：O(1) - 只用几个变量
  - n = 20000 时，约 2万次运算

性能：备忘录法和双指针法都比暴力法快 10000+ 倍！

def trap(height: list[int]) -> int:
    """接雨水 - 包含边界情况处理"""

        if height[left] < height[right]:
            if height[left] >= left_max:
                left_max = height[left]
            else:
                water += left_max - height[left]
            left += 1
        else:
            if height[right] >= right_max:
                right_max = height[right]
            else:
                water += right_max - height[right]
            right -= 1
    
    return water

## 五. 小贴士

### 1. 为什么不能接雨水的位置要跳过？

```
边界柱子不能接雨水的原因：

位置0：左边没有柱子，无法形成容器
       ┌─ 无法形成左边界
       ↓
  █    █ █
  █    █ █
  ████████████
  0 1 2 3 4

解决方案：
  - 只遍历中间位置（跳过首尾）
  - 或者在算法中处理（双指针法天然跳过）
```

### 2. 核心公式详解

```
每个位置能接的雨水 = min(左边最高, 右边最高) - 当前高度

为什么是 min 而不是 max？

例子：
  █
  █     █
  █  █  █
  ███  ███
  
  左边最高 = 3
  右边最高 = 3
  中间高度 = 1
  
  如果用 max(3, 3) - 1 = 2 → 错误！（最多只能接2单位水，但实际只有1单位空间）
  如果用 min(3, 3) - 1 = 2 → 正确！

因为：水流会从较矮的边界溢出，较矮的边界决定了水位！
```

### 3. 进阶思考：如何扩展到多行？

```
问题：如果是二维的矩阵，如何计算能接的雨水？

解决方案：
  1. 逐行计算（类似本题）
  2. 使用最小堆（类似 Dijkstra）
  3. 多源 BFS

本题是二维问题在一维上的简化！
```

## 六. 完整代码实现

### Python（双指针法 - 最优解）

```python
from typing import List

class Solution:
    def trap(self, height: List[int]) -> int:
        """
        接雨水 - 双指针法
        
        时间复杂度：O(n)
        空间复杂度：O(1)
        
        算法步骤：
            1. 初始化左右指针和最大高度
            2. 比较左右指针指向的柱子高度
            3. 处理较矮的一边，计算雨水
            4. 移动指针，继续处理
        
        Args:
            height: 表示柱子高度的数组
            
        Returns:
            int：能接住的雨水总量
        """
        n = len(height)
        
        # 边界情况
        if n == 0:
            return 0
        
        # 初始化指针和变量
        left = 0
        right = n - 1
        left_max = 0
        right_max = 0
        water = 0
        
        # 双指针遍历
        while left < right:
            # 如果左边柱子较矮，处理左边
            if height[left] < height[right]:
                # 更新左边最大高度
                if height[left] >= left_max:
                    left_max = height[left]
                else:
                    # 可以接雨水
                    water += left_max - height[left]
                left += 1
            else:
                # 如果右边柱子较矮或相等，处理右边
                if height[right] >= right_max:
                    right_max = height[right]
                else:
                    # 可以接雨水
                    water += right_max - height[right]
                right -= 1
        
        return water
```

### Python（暴力法 - 容易理解）

```python
from typing import List

class Solution:
    def trap(self, height: List[int]) -> int:
        """
        接雨水 - 暴力法
        
        时间复杂度：O(n²)
        空间复杂度：O(1)
        """
        n = len(height)
        water = 0
        
        for i in range(1, n - 1):
            # 找左边最高
            left_max = 0
            for j in range(i):
                left_max = max(left_max, height[j])
            
            # 找右边最高
            right_max = 0
            for j in range(i + 1, n):
                right_max = max(right_max, height[j])
            
            # 加上当前位置的雨水
            water += min(left_max, right_max) - height[i]
        
        return water
```

### C++（双指针法）

```cpp
#include <vector>

class Solution {
public:
    /**
     * 接雨水 - 双指针法
     * 
     * 时间复杂度：O(n)
     * 空间复杂度：O(1)
     */
    int trap(std::vector<int>& height) {
        int n = height.size();
        
        if (n == 0) {
            return 0;
        }
        
        int left = 0;
        int right = n - 1;
        int left_max = 0;
        int right_max = 0;
        int water = 0;
        
        while (left < right) {
            if (height[left] < height[right]) {
                if (height[left] >= left_max) {
                    left_max = height[left];
                } else {
                    water += left_max - height[left];
                }
                left++;
            } else {
                if (height[right] >= right_max) {
                    right_max = height[right];
                } else {
                    water += right_max - height[right];
                }
                right--;
            }
        }
        
        return water;
    }
};
```

## 八. 总结

### 核心要点

1. **核心公式**：`min(左边最高, 右边最高) - 当前高度`
2. **备忘录优化**：预计算左右最高柱子，O(n) 时间
3. **双指针优化**：只关心较小边界，O(1) 空间
4. **边界处理**：跳过首尾位置

### 算法模板

```python
def trap(height):
    if not height or len(height) < 3:
        return 0
    
    left, right = 0, len(height) - 1
    left_max, right_max = 0, 0
    water = 0
    
    while left < right:
        if height[left] < height[right]:
            if height[left] >= left_max:
                left_max = height[left]
            else:
                water += left_max - height[left]
            left += 1
        else:
            if height[right] >= right_max:
                right_max = height[right]
            else:
                water += right_max - height[right]
            right -= 1
    
    return water
```
